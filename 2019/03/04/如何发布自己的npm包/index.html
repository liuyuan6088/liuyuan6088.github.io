如何发布自己的npm包

发表于 2019-03-04 | 分类于 npm   |   热度  °C | | 阅读次数:

前沿

最近做项目，遇到一个较多数据展示的一个页面，想有一个数字上下滚动的效果，于是去往上搜了一下，
好像并没要基于react的web版这样的三方库 =0=，无赖只能自己实现一个组件，顺便做成一个npm包，
之前没有发布过npm包，可能会对NPM对开发有一种蜜汁敬畏，觉得这是一个很高大上的东西。其实，在现在的我看来，npm包就是一个我们平时经常写的一个export出来的模块而已，只不过跟其它业务代码耦合性低，具有较高的独立性。 当然，要发布一个npm包，除了写的模块组件外，还需要做一些基础的包装工作。

创建仓库

在github上新建一个仓库，名字可以自己取。确保你创建的组件名称没有在 npm 上被使用过，
然后克隆项目，初始化：

git clone https://github.com/xxx/wsm-loading.git
cd wsm-loading
npm init

运行npm init问题提示列表可以根据自己的个人爱好填写，也可以采取默认的选项

  • name：
    模块名，之后发布之后就可以让用户npm install xxxx来引用你的开源模块了
  • version：
    版本号，语义版本号分为X.Y.Z三位，分别代表主版本号、次版本号和补丁版本号。当代码变更时，版本号按以下原则更新。
    1. 如果只是修复bug，需要更新Z位。
    2. 如果是新增了功能，但是向下兼容，需要更新Y位。
    3. 如果有大变动，向下不兼容，需要更新X位。
  • description：
    简单介绍自己的模块
  • main：
    入口文件，必要，当通过require(‘xxx’)时，是读取main里声明的文件
  • test command：
    测试命令
  • git repository：
    git仓库地址
  • keywords：
    关键词，可以通过npm搜索你填写的关键词找到你的模块
  • author：
    作者信息，可以之后编辑更详细一些
  • license（ISC）：
    代码授权许可 可以参考这里

因为是基于react，所以先安装react相关的，或者使用官方脚手架：

目录结构

关键的组件代码，思路是组件接受两个属性，一个是需要展示的数字，一个是类型，当前后的数字不一样，将执行滚动动画，type类型有3种：普通数字，金额，百分比，不同类型需做不同的格式化。而关键的上下滚动效果，思路是每一个数字都是是个列表项，每个列表包含0-9上下排列的子元素，但每个列表项的可视窗口只有一个子元素的高度，通过定位的top属性去滚动数字，已到达数字上下滚动的效果（最开始用的是css的过度实现，但发现当距离过短，比如1-2，1-3的变化，动画不流畅，所以又使用咯requestAnimationFrame实现，但还是有点问题是时间不能统一，而且不能像贝塞尔曲线那般好控制，后续会针对这个在改改吧）
